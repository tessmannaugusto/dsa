# Big O Notation
- maneira de estabelecer o quão rápido é um algoritmo ao analisar o número de operações que um algoritmo realiza.
- estabelece o tempo de execução para a pior hipótese.
- O(n) - tempo de execução conhecido como tempo linear. Ex: pesquisa simples (acessa todos)
- O(log n) - tempo de execução logarítimico, ex: pesquisa binária
- O(n * log n) - algoritmo rapido de ordenção, como o quicksort. 
- O(n2) - algoritmo lento de ordenação, como a ordenação por seleção
- O(n!) - algoritmo lento, como o do caixeiro viajante
## pontos principais
- a rapidez de um algoritmo não é medida em segundos, mas sim pelo crescimento do numero de operações
